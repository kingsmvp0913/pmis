# PMIS 營造廠商報表系統 — 設計書

- **日期**:2026-07-02
- **狀態**:設計待審閱
- **專案路徑**:`C:\pmis\`
- **平台**:僅 Windows

---

## 1. 系統目的

PMIS 是一套管理營造廠商、並把各廠商繳交的文件自動轉成「固定格式營造報表」的本機系統。

使用者是**完全不懂程式的人**,因此:
- 全程中文、畫面引導
- **點兩下就能安裝 / 啟動**(用 `.bat`,不是 PowerShell)
- 所有環境自動設定好,資料自動存進資料庫

核心價值:一次設定好「廠商 × 報表」的對應規則後,日後**只要把廠商檔案丟進某報表的區塊,就自動產出該報表**。

---

## 2. 使用者心智模型(三種可管理的東西)

1. **廠商(Vendor)** — 使用者自行建立(廠商A、廠商B…),含名稱、統一編號等,並設定「識別方式」讓系統日後能自動認出這家。
2. **報表(Report)** — 使用者自行建立(報表A、報表B…),每個報表上傳一份**固定範本**(輸出長相),系統自動從範本抽出「這個報表需要哪些欄位」。
3. **對應設定(Mapping)** — 針對「**某廠商 × 某報表**」設定一次:廠商範本欄位 → 報表欄位。系統自動建議,沒對到的欄位**標紅字**讓使用者手動設定。

---

## 3. 操作流程

### 3.1 一次性設定(3 步)

1. **建立廠商**:填廠商名稱、統一編號等基本資料。
2. **建立報表**:上傳固定報表範本(如 `營造報表A.xlsx`)→ 系統自動抽出報表所需欄位清單。
3. **設定廠商 × 報表**:
   - 上傳該廠商的來源範本 → 系統自動判別格式並抽出欄位。
   - 系統**自動建議對應**(依欄名相似度);未對到的欄位**標紅字**,使用者手動指定「這欄 = 報表的哪個欄位」。
   - 設定**識別方式**(例:統一編號值、廠商名稱出現在某儲存格、檔名關鍵字),供日後自動辨識。

### 3.2 日常操作(全自動)

```
把廠商檔案 → 丟進「報表A 區塊」
   ↓ 系統自動判別格式、讀出資料
   ↓ 依「識別方式」自動辨識是哪家廠商
   ↓ 找到該廠商 × 報表A 的對應設定
   ↓ 依對應把資料填入報表A 範本
產出報表A(存到 data/output/)✅
```

**辨識失敗處理**:若系統認不出廠商(無符合/多筆符合),**跳出清單請使用者點選確認**,絕不硬猜出錯。

## 4. 支援的檔案格式

**來源檔案(廠商範本)可讀取格式:**

| 類別 | 格式 | Python 解析套件 |
|---|---|---|
| 試算表 | Excel `.xlsx` `.xls` | `openpyxl` / `pandas` |
| 試算表 | CSV `.csv` | `pandas` |
| 試算表 | ODF 試算表 `.ods` | `odfpy` / `pandas` |
| 文件 | Word `.docx` | `python-docx` |
| 文件 | 文字型 PDF `.pdf` | `pdfplumber` |

**輸出報表範本**:Excel(`openpyxl` 填值)或 Word(`python-docx` / `docxtpl` 填值),依使用者提供的範本類型而定。

**明確不做(範圍外)**:
- 圖片(JPG/PNG/TIFF)與掃描版 PDF 的 **OCR 辨識** — 使用者確認辦公室極少用,移除以保持系統穩定。
- PowerPoint 作為資料來源 — 罕見,暫不納入;未來如有需要再以 `python-pptx` 擴充。

## 5. 技術架構

- **語言**:Python(使用者電腦已安裝);安裝時建立獨立虛擬環境(venv),不污染系統。
- **介面**:本機小型網頁伺服器(FastAPI)+ 瀏覽器操作,體驗與 odoo-v2 一致(點兩下 → 自動開瀏覽器)。
- **資料庫**:**SQLite**(單一檔 `data/pmis.db`)。零安裝、零設定,完全符合「資料存資料庫就好」。所有操作走畫面,使用者不需寫 SQL。
- **服務埠**:`localhost:4141`(避開 odoo-v2 的 3939)。

### 5.1 點兩下的檔案

| 檔名 | 用途 | 行為 |
|---|---|---|
| `安裝.bat` | 第一次安裝 | 顯示進度視窗 → 檢查 Python → 建 venv → `pip install` → 初始化 SQLite → 開啟設定頁 |
| `啟動.bat` | 每天使用 | 背景啟動伺服器 → 自動開瀏覽器 |

- 兩者皆為 `.bat`(一般批次檔,非 PowerShell),符合使用者要求。
- 安裝時顯示進度讓使用者安心;每天啟動則安靜背景執行、直接彈出瀏覽器。

## 6. 元件職責(單一職責、可獨立測試)

| 元件 | 職責 | 輸入 → 輸出 |
|---|---|---|
| `parsers/base.py` | 依副檔名 + 內容判別格式,分派給對應讀取器 | 檔案路徑 → 統一結構(欄位清單 + 資料列) |
| `parsers/*.py` | 各格式讀取器,回傳統一結構 | 檔案 → `{fields:[...], rows:[...]}` |
| `mapping.py` | 比對來源欄位 vs 報表欄位,產生建議對應;儲存/讀取對應表 | 來源欄位 + 報表欄位 → 建議對應 + 未對應清單 |
| `identify.py` | 依廠商識別規則,判斷一份來源檔屬於哪家廠商 | 檔案內容 + 規則庫 → 廠商(或「需人工確認」) |
| `report.py` | 依對應把資料填入報表範本,產出檔案 | 資料 + 對應 + 報表範本 → 產出報表檔 |
| `db.py` | SQLite 資料存取 | — |

各讀取器對外統一介面,新增格式不影響其他元件。

## 7. 資料庫設計(SQLite)

| 資料表 | 主要欄位 | 說明 |
|---|---|---|
| `vendors` | id, name, tax_id, note, created_at | 廠商 |
| `vendor_signatures` | id, vendor_id, rule_type, rule_value | 廠商識別規則(可多條:統編/名稱位置/檔名關鍵字) |
| `reports` | id, name, template_path, template_type, created_at | 報表(含固定範本) |
| `report_fields` | id, report_id, field_name, location, sort | 報表所需欄位(從範本抽出) |
| `vendor_templates` | id, vendor_id, report_id, source_filename, source_type, uploaded_at | 某廠商為某報表上傳的來源範本 |
| `source_fields` | id, vendor_template_id, field_name, sample_value, location | 來源範本抽出的欄位 |
| `mappings` | id, vendor_id, report_id, source_field, report_field | 對應表(per 廠商 × 報表) |
| `outputs` | id, vendor_id, report_id, source_file, output_path, status, warnings, created_at | 產出記錄 |

## 8. 錯誤處理

- **格式無法解析** → 明確中文錯誤訊息,標示偵測到的檔案類型,不中斷其他流程。
- **廠商辨識失敗 / 多筆符合** → 跳出清單請使用者點選確認。
- **對應缺漏(仍有紅字未設定)** → 阻擋產出並提示補齊;或允許部分產出並明確警告(產出記錄標 `warnings`)。
- **報表範本欄位對不到來源資料** → 該欄留空並在產出記錄標註,不靜默略過。

---

## 9. 測試策略

- **讀取器單元測試**:每種格式(Excel/CSV/ODS/Word/PDF)用真實樣本檔驗證,確認抽出的欄位與資料正確。
- **對應建議測試**:給定來源欄位與報表欄位,驗證建議結果與未對應清單。
- **辨識測試**:多家廠商規則下,驗證能正確辨識 / 正確回報「需人工確認」。
- **端對端測試**:上傳樣本檔 → 產出報表 → 比對內容正確。
- 測試須驗證「為什麼」(Rule 9):例如辨識規則衝突時必須回報人工確認,而非任選一家。

---

## 10. 範圍界線(YAGNI)

**納入**:廠商/報表/對應/自動辨識/產出、Excel/CSV/ODS/Word/文字PDF 讀取、安裝與啟動檔、SQLite 儲存。

**不納入**:圖片與掃描件 OCR、PowerPoint 來源、多人連線 / 帳號權限(單機單人使用)、雲端部署、使用者自寫 SQL 查詢介面。

---

## 11. 待使用者於實作前提供

- **固定報表範本檔**(至少一份,用來抽出報表欄位並作為輸出格式)。
- **一份廠商來源範本樣本**(用來驗證讀取與對應)。

## 12. 追加功能(依使用者需求,實作於初版之後)

建築師事務所行政助理提出後追加的實用功能。其中「多人帳號權限」推翻了 §10 原本
「單機單人、不做帳號權限」的界線 —— 這是使用者的明確要求。

1. **多家廠商彙總成一張總表**(`aggregate.py`):一次上傳多個廠商檔,各自辨識 +
   套用對應,填入「表頭式 Excel 範本」成單一總表;無法自動辨識的檔案會列出略過。
2. **交件追蹤 / 缺件提醒**(`report_vendors` / `submissions`):每個報表可設定
   「應交廠商」清單,依期別顯示該交 / 已交 / 未交。產出時自動登記交件。
3. **PDF 匯出與自動命名歸檔**(`pdf_export.py`):產出後可轉 PDF(優先用 LibreOffice
   headless,退而用 docx2pdf);檔名自動用「報表_廠商_期別_時間」規則。
4. **數字驗算**(`validate.py`):報表欄位可標記為「金額」,彙總時自動算總計、
   稅額(5%)、含稅總計;單筆多列時提供逐列加總對照。
5. **上期資料帶入**(`outputs.values_json` + `report.generate(carry_over=...)`):
   產出時可帶入同報表同廠商上一次的固定欄位值,本次留空者自動填入。
6. **報表範本改版保留舊版**(`report_template_versions`):上傳新範本 → 版本 +1,
   保留歷史;每筆產出記錄使用的範本版本。

**多人帳號與權限**(`auth.py`、`users` / `user_permissions`):
- 密碼以 PBKDF2-HMAC-SHA256 雜湊(標準函式庫,不需額外套件)。
- 角色 admin(全功能,含使用者管理)/ staff(僅限被開放的功能)。
- 管理者可建立同事帳號,並逐項勾選開放的功能(管理廠商 / 報表 / 對應 /
  產出 / 產出記錄 / 交件追蹤 / 使用者管理)。
- 首次啟動自動建立預設管理者 `admin` / `admin`,並提示盡快改密碼。
- Session 以簽章 Cookie 保存,金鑰持久化於 `data/.session_secret`。

仍**不納入**:圖片與掃描件 OCR、PowerPoint 來源、雲端部署、使用者自寫 SQL 介面。
