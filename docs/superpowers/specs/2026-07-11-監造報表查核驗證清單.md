## 欄位標記說明
- **資料來源**：
  - `【日誌】`＝可由營造廠商施工日誌（附表四）自動抽取。
  - `【估驗】`＝由估驗計價單／詳細表帶入（完成數量×契約單價）。
  - `【主檔】`＝工程主檔／契約帶入（契約數量、契約金額、預定進度網圖）。
  - `【監造判斷】`＝監造方現場查核後人工判定，無法自動產生。
- **可否自動勾稽**：
  - `🟢 可自動`＝數字比對／邏輯運算即可判定（系統可跑）。
  - `🟡 半自動`＝系統可算出結果並標記異常，但「是否可接受」須人工複核。
  - `🔴 需人工`＝屬現場專業判斷，系統僅能提示待辦、無法判定對錯。

## A. 數量類

| # | 查核項目 | 資料來源 | 自動勾稽 |
|---|---|---|---|
| A1 | 施工日誌「本日完成數量」加總 = 累計完成數量（逐日累加無跳號/漏日） | 【日誌】 | 🟢 可自動（本日Σ vs 累計欄） |
| A2 | 累計完成數量 = 估驗計價單之計價數量（同一工項數量一致） | 【日誌】↔【估驗】 | 🟢 可自動（同工項比對，容差=0 或契約約定） |
| A3 | 累計完成數量 ≤ 契約數量（未超出契約） | 【日誌／估驗】↔【主檔】 | 🟢 可自動（>契約即警示，屬須辦契變之訊號） |
| A4 | 各工項完成數量單位與契約詳細表單位一致（m³/m²/式…） | 【日誌】↔【主檔】 | 🟢 可自動（單位字串比對） |
| A5 | 材料本日使用量、累計使用量之合理性（用量 vs 完成量比例） | 【日誌】 | 🟡 半自動（可算比例，異常門檻須人工設定） |
| A6 | 超出契約數量之工項是否已辦理契約變更 | 【監造判斷】↔【主檔】 | 🟡 半自動（系統可標「超量未變更」，核准與否人工） |

## B. 金額類

| # | 查核項目 | 資料來源 | 自動勾稽 |
|---|---|---|---|
| B1 | 估驗計價金額 = Σ(本期計價數量 × 契約單價)（重算核對） | 【估驗】↔【主檔】 | 🟢 可自動（重算，須用四捨五入 ROUND_HALF_UP，非銀行家捨入） |
| B2 | 累計估驗金額 ≤ 契約金額（或變更後契約金額） | 【估驗】↔【主檔】 | 🟢 可自動（>即超額警示） |
| B3 | 累計完成金額 / 契約金額 ≈ 實際進度%（金額進度與百分比進度相符） | 【估驗】↔【日誌】 | 🟢 可自動（差距超容差即標記） |
| B4 | 保留款計算正確（依契約比例，如估驗款5%~10%） | 【估驗】↔【主檔】 | 🟢 可自動（按契約保留比例重算） |
| B5 | 本期估驗淨付款 = 本期估驗額 − 保留款 − 應扣款（預付款回收等） | 【估驗】↔【主檔】 | 🟢 可自動（公式重算） |
| B6 | 變更後契約金額 = 原契約 ± 各次契變金額 | 【主檔】 | 🟢 可自動（契變累加核對） |
| B7 | 完成金額與實際施工進度是否相符（有無灌水/提前計價） | 【監造判斷】 | 🔴 需人工（是否確實完成屬現場判斷） |

## C. 進度類

| # | 查核項目 | 資料來源 | 自動勾稽 |
|---|---|---|---|
| C1 | 實際進度% vs 預定進度%（差值＝落後/超前） | 【日誌】↔【主檔】 | 🟢 可自動（實際−預定，負值為落後） |
| C2 | 落後判定（落後 > 契約門檻，如落後10%須提改善措施） | 【日誌】↔【主檔】 | 🟢 可自動（門檻比較，門檻值人工設定） |
| C3 | 累計實際進度 = 各工項(完成量/契約量 × 權重)加總（重算 S-Curve） | 【日誌】↔【主檔】 | 🟢 可自動（依預定進度表權重重算） |
| C4 | 工期核對：累計施工天數 = 開工日→報表日之日曆天（扣不計工期） | 【日誌】↔【主檔】 | 🟢 可自動（日期差重算） |
| C5 | 剩餘工期 = 契約工期(含展延) − 累計天數 ≥ 0 | 【主檔】 | 🟢 可自動（≤0 為逾期警示） |
| C6 | 預定/實際進度是否於每周六及月底估填（填報頻率合規，S5） | 【監造判斷】 | 🟡 半自動（系統可查有無填、日期是否落在規定日） |
| C7 | 契變/展延後進度是否已改依「變更後進度網圖」計算 | 【監造判斷】↔【主檔】 | 🟡 半自動（系統可提示須換基準，換算正確與否人工） |

|---|---|---|---|

| D3 | 施工抽查合格率（本日／累計）＝ 合格件數/抽查件數 | 【監造判斷】 | 🟡 半自動（有抽查明細則可算比率，合格與否人工判） |

| D7 | 工地職安衛督導：施工前檢查□完成/□未完成 | 【監造判斷】 | 🔴 需人工（現場勾選） |

| # | 查核項目 | 資料來源 | 自動勾稽 |
|---|---|---|---|
| E1 | **施工日誌 ↔ 監造報表 ↔ 估驗單三者「同工項完成數量」一致** | 【日誌】↔監造↔【估驗】 | 🟢 可自動（核心勾稽，不一致即攔） |
| E2 | 三表「累計完成金額／進度%」一致 | 【日誌】↔【估驗】 | 🟢 可自動（差距超容差標記） |
| E3 | 日期一致：施工日誌日期 = 監造報表填報日期（同日對應） | 【日誌】↔監造 | 🟢 可自動（日期比對，缺對應日即漏報） |
| E4 | 天氣合理性：施工日誌天氣 = 監造報表天氣（上午/下午一致） | 【日誌】↔監造 | 🟢 可自動（同來源應相同；雨天卻報大量戶外完成量→提示） |
| E5 | 停工/雨天邏輯：報停工日卻有完成數量／計價，或晴天連續零產出 | 【日誌】 | 🟡 半自動（可標記矛盾，是否合理人工判） |

| E7 | 表報流水編號連續、無跳號/重號 | 監造 | 🟢 可自動（序號連續性檢查） |
| E8 | 契約基準一致：三表引用之契約金額/工期/數量皆為同一（最新變更後）版本 | 【主檔】 | 🟢 可自動（版本號/契變次數比對） |

## F. 最重要「可自動勾稽」項目（優先實作，🟢）

1. **E1 — 施工日誌↔監造報表↔估驗單「同工項完成數量」三表一致**（系統核心，不一致即攔）。
2. **A2/A3 — 累計完成數量 = 估驗計價數量，且 ≤ 契約數量**（超量即警示須契變）。
3. **B1/B2 — 估驗金額 = Σ(數量×單價) 重算相符，且累計 ≤ 契約金額**（用 ROUND_HALF_UP）。
4. **B3 — 累計完成金額/契約金額 ≈ 實際進度%**（金額進度與百分比進度勾稽）。
5. **C1/C2 — 實際進度% − 預定進度% = 落後值，落後超門檻自動示警**。
6. **B4/B5 — 保留款與本期淨付款依契約比例重算相符**。
7. **C4/C5 — 累計施工天數、剩餘工期重算，逾期自動示警**。
8. **E3/E7 — 日誌與監造報表日期對應無漏日、表報編號連續無跳號**。

> **無法自動、只能列人工待辦者：** D1 材料規格相符、D5 按圖施工、D7 職安衛督導（現場勾選）、B7 完成金額是否確實反映實際施工——皆屬監造方現場專業判斷，系統只能提示待辦與提供比對數字，不得替代判定（🔴）。抽查合格率（D3）、抽驗合格（D2/D4）、缺失追蹤（D6）為半自動：系統算比率/追狀態，合格認定人工（🟡）。
