# 監造報表公版範本盤點

> **用途：** 為 PMIS「監造報表產生功能」蒐集可下載的公版範本，作為產出格式與欄位比對之依據。
> **建立日期：2026-07-11。**
> 交叉引用：[`2026-07-11-監造報表目標欄位與規則(草案).md`](2026-07-11-監造報表目標欄位與規則(草案).md)（目標欄位 schema）、[`2026-07-11-監造報表查核驗證清單.md`](2026-07-11-監造報表查核驗證清單.md)（查核清單）。
>
> **重要結論（Excel 可用性）：** 中央（工程會）及各縣市之**「監造報表」本體公版一律為 Word/ODT/PDF，並無官方 Excel（.xls/.xlsx）版本**。唯一取得到的官方 Excel 是雲林縣府頒的**「預定進度表」（.xls）**——它是與監造報表配套、承載「預定進度% vs 實際進度%」數字的表，對本系統的進度勾稽最有價值。監造報表本體若需 Excel，須自行以工程會附表五欄位重製。

## 2. 最推薦作為「產出範本」的檔案

### 監造報表本體 → **T4 `監造日報表_臺北市.docx`（直轄市加值版，欄位最完整）**
- 為何選它：以工程會附表五為底稿，額外含 **契約編號、主辦機關、第一聯/第二聯**，且註記明確要求「**一份併估驗詳細表送核**」「**預定進度與實際進度應於每周六及月底估計填寫一次**」——與本系統「估驗↔進度勾稽」需求高度契合，且是可直接改樣式的 DOCX。
- 若要中央法定底稿（全國通用、無縣市加欄）→ 用 **T1 `監造報表_工程會附表五.odt`**（工程會版，權威性最高）。

**T4 臺北市監造日報表主要欄位（擷取自 document.xml）：**
- 表頭：監造單位全銜、公共工程監造日報表、本日天氣（上午／下午）、填報日期（年月日星期）、工程名稱、契約編號、主辦機關、契約工期、開工日期、預定竣工日期、實際竣工日期、契約變更次數、工期展延天數、契約金額（原契約／變更後契約）、**預定進度(%)**、**實際進度(%)**。
- 本體五大項（監造查核）：
  1. 工程進行情況（含約定之重要施工項目及數量）
  2. 監督依照設計圖說及核定施工圖說施工（含檢驗停留點及施工抽查等情形）
  3. 查核材料規格及品質（含檢驗停留點、材料設備管制及檢（試）驗等抽驗情形）
  4. 督導工地職業安全衛生事項（施工前檢查辦理情形□完成□未完成；其他工安督導事項）
  5. 其他約定監造事項（重要事項紀錄、主辦機關指示及通知廠商辦理事項等）
- 簽章：監造單位簽章。
- 關鍵註記：契變/工期展延後應填修正核定後契約工期與預定竣工日期，並依修正核定後進度網圖計算累計預定/實際進度；工程項目依契約詳細表選重要項目填入；本表不得塗改。

### 進度數字表 → **T7 `預定進度表_雲林縣.xls`（唯一官方 Excel）**
- 主要欄位（xlrd 擷取，Big5 編碼）：**項次、工程項目、工程金額、百分比（權重）、預定進度**（按月分欄，10、20、30…百分比累計格）。
- 用途：這是承載「預定進度%（S-Curve 分母）」的 Excel，是監造報表「預定進度 vs 實際進度」勾稽的來源之一。

> **一致性結論：** 四份監造報表本體欄位**高度一致**（皆以工程會附表五五大項為骨幹），縣市差異僅在「加欄」（臺北市加契約編號/主辦機關/聯別）與註記細節。因此 PMIS 以工程會附表五為 schema 底稿、臺北市加值欄為選配，即可涵蓋主流縣市（與草案文件第 2 節一致）。

## 4. Excel 可用性總結（回應任務一核心問題）

- **監造報表本體：找不到官方 Excel。** 工程會（ODT/PDF）、臺北市（DOCX）、高雄市（ODT）、雲林縣（DOC）皆非 Excel。原因：這些是「文字敘述式查核表」（五大項為長文欄），非表格計算用途，政府一律以 Word/ODT 發布。
- **配套數字表：有官方 Excel。** 雲林縣「預定進度表」為 .xls（T7），是唯一取得的官方 Excel，正好承載進度百分比數字。
- **建議：** 若 PMIS 要以 Excel 產出監造報表，須自行以工程會附表五（T1）欄位重製 .xlsx 樣板（置於 `<module>/static/xls/` 依專案規範）；「表格化數字」（完成數量/金額/進度）可參考 T7 的欄位結構。
